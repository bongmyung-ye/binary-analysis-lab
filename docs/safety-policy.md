# Safety Policy

This repository is designed for defensive reverse engineering practice and binary analysis documentation.

## Allowed scope

- Self-built sample programs included in this repository
- Public training material that explicitly permits analysis
- Local analysis environments owned by the researcher
- Static analysis, dynamic debugging, string review, hashing, and report writing
- YARA rules written for safe samples in this repository

## Out of scope

- Malware distribution or execution outside an isolated lab
- Unauthorized analysis of third-party commercial software
- Cracking, license bypassing, DRM removal, or cheat development
- Credential theft, persistence, evasion, or destructive behavior
- Instructions for compromising real systems or accounts

## Sample policy

The sample programs in this repository must stay small, safe, and transparent. They are used to demonstrate analysis workflow, not to simulate harmful behavior.

Each sample should include:

- Source code
- Build notes
- Analysis goal
- Expected observations
- A short report

## Reporting style

Reports should describe what was observed and how the conclusion was reached. They should not provide instructions for modifying third-party binaries or bypassing protections.
